# Отчет по реализации генератора бесшовных текстур

В рамках тестового задания была исследована задача генерации текстур высокого разрешения. Был проведен анализ классических CV-методов, специализированных GAN-архитектур и современных Diffusion-пайплайнов.

В качестве **основного решения** выбран подход на базе MMDiT (Nano Banana Pro) с предварительной генерацией промпта, так как он обеспечивает наилучшее качество «из коробки» при минимальных затратах на настройку пайплайна.

---

## 1. Анализ альтернативных подходов

В ходе исследования были рассмотрены, но отложены следующие методы:

### A. Classical CV: Image Quilting
В качестве классического baseline рассматривался алгоритм **Image Quilting**.
*   **Статус:** Отклонено.
*   **Причина:** Доступные Open Source реализации алгоритма не обновлялись более 5 лет. Отсутствие фиксации зависимостей и устаревший стек требуют значительного времени на подбор версий Python и библиотек для компиляции и запуска. В условиях ограниченного времени было решено сосредоточиться на более актуальных DL-методах.

### B. GAN-based Approaches
Рассматривалась возможность использования генеративно-состязательных сетей для быстрого инференса.
*   **Статус:** Отклонено.
*   **Причина:** Отсутствие Foundation-моделей в семействе GAN, пригодных для zero-shot генерации произвольных текстур.
    1.  **[tileGAN](https://github.com/afruehstueck/tileGAN):** Отсутствуют базовые веса, в доступе только стилизованные модели, что ограничивает домен применения.
    2.  **[InfinityGAN](https://github.com/hubert0527/infinityGAN):** Архитектурно подходит для задачи, но публичные веса обучены на датасете Flickr (пейзажи/природа).

---

## 2. Реализованное решение

В качестве рабочего пайплайна выбрана генерация с использованием MMDiT модели **Nano Banana Pro** в связке с LLM для улучшения промпт conditioning.

### Архитектура решения
1.  **Prompt Engineering:** Исходное описание текстуры (или референс) сначала обрабатывается **Gemini 3 Flash**. Модель генерирует детализированный промпт, описывающий структуру, материалы и особенности поверхности. Это выступает как улучшенный condition для генератора.
2.  **Image Generation:** Сгенерированный промпт подается в **Nano Banana Pro**. Изображение генерируется кусками в максимально доступном разрешении 4K (`4096x4096`). Для сохранения консистентности текстуры при генерации новых участков используется overlap с уже сгенерированной текстурой.

> **Note:** В ходе экспериментов также тестировался **Flux 2 Pro**. Несмотря на высокую детализацию, модель показала склонность к искажению цветовой палитры относительно ожидаемого результата, поэтому выбор был сделан в пользу Nano Banana.

### Преимущества подхода
*   **SOTA качество:** Высокая детализация и реалистичность материалов.
*   **Простота интеграции:** Отсутствие необходимости обучать или дообучать модель с нуля.
*   **Семантический контроль:** Использование LLM позволяет точно управлять стилем текстуры через текст.

## 3. Установка и запуск

Для управления зависимостями и виртуальным окружением в проекте используется [uv](https://github.com/astral-sh/uv).

### 1. Клонирование и установка зависимостей

```bash
git clone https://github.com/LightCooling/VQTEK-apply.git
cd VQTEK-apply
uv sync
```

### 2. Настройка API ключа

Для работы пайплайна необходим доступ к LLM через OpenRouter. Создайте файл `key.txt` в корневой директории проекта и поместите туда ваш [API ключ](https://openrouter.ai/settings/keys) (без лишних пробелов и переносов строк).

```bash
# Пример создания файла (Linux/macOS)
echo "sk-or-v1-your-key-here" > key.txt
```

### 3. Запуск генерации

Запуск основного скрипта выполняется командой:

```bash
python edit.py -i 89395115.png -o test.png -H 4200 -W 4200
```

**Аргументы:**
*   `-i`: Путь к входному изображению (референс или исходная текстура).
*   `-o`: Имя выходного файла.
*   `-H`: Целевая высота (Height).
*   `-W`: Целевая ширина (Width).

## 4. Ограничения и планы по улучшению

Текущая реализация является MVP и обладает рядом ограничений, которые могут быть устранены при дальнейшей работе над проектом:

### Текущие ограничения
1.  **Стоимость и артефакты:** Использование проприетарной модели (Nano Banana Pro) влечет расходы на генерацию. Также модель добавляет невидимые водяные знаки, и возможные микро-искажения цветов.

### Возможные улучшения
При наличии дополнительного времени целесообразно рассмотреть специализированные диффузионные пайплайны:

*   **Tiled Diffusion (MultiDiffusion):**
    Использование подхода [Tiled Diffusion](https://github.com/madaror/tiled-diffusion) позволяет генерировать изображения произвольного размера с нативной поддержкой бесшовности на стыках тайлов.
*   **Open Source Foundation Models + LoRA:**
    Переход на открытые модели (Flux.1 и 2, SDXL) с дообучением LoRA адаптеров специально под задачу "texture outpainting". Это решит проблему стоимости генерации и даст полный контроль над цветопередачей и масками диффузии.

### Рассмотренные альтернативы

Код **Tiled Diffusion** был успешно запущен, текстура сгенерирована. Однако, в примерах рассматривается только генерация "с нуля" или замена произвольного шума на исходное изображение.
Таким образом, полученное изображение является лишь имитацией "на тему" и в основном ориентируется на предоставленный промпт.
Однако, сам подход гибок и позволяет организовать полноценный тайлинг с сохранением исходного изображени. Данный метод не был реализован практически в связи с его трудозатратностью.
